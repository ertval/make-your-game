# 🚀 test(A-06): unskip and implement render desync e2e tests using getWorld hook

> **Summary**: Unskips and implements three Playwright E2E tests (`#84`, `#85`, `#104`) in `tests/e2e/render-desync-bugs.spec.js` using the exposed ECS `getWorld()` runtime test hook.

---

## 📝 Description

### 🔄 What Changed
- **`tests/e2e/render-desync-bugs.spec.js`**: Re-implemented and unskipped `#84` (Bomb rendering), `#85` (Destructible wall cell removal), and `#104` (Pellet removal convergence). Used coordinate-based selectors instead of dynamic cell classes to prevent false positive flaking.
- **`docs/implementation/track-a.md`**: Updated `A-06` deliverables to record the completed E2E unskip check.
- **`docs/implementation/audit-traceability-matrix.md`**: Updated the Last modified timestamp to keep verification synchronized.

### 🎯 Why
- **Rationale**: The E2E tests were previously skipped because they lacked a clean runtime hook to inspect/mutate the ECS world directly. Now that the `getWorld()` hook exists, we can directly verify that DOM sync behaves correctly on map mutations and bomb/collider mutations without relying on fragile keyboard timings.
- **Impact**: Increased test suite reliability and coverage for rendering and board-sync desync issues.
