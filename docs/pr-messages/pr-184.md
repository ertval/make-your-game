# 🚀 test(A-06): align DOM budget assertions with AGENTS.md limit

> **Summary**: Enforces the ≤500 DOM elements budget from `AGENTS.md` by updating E2E tests and reducing the pre-allocated bomb/fire sprite pool capacities.

---

## 📝 Description

### 🔄 What Changed
- **`src/ecs/resources/constants.js`**: Reduced `POOL_MAX_BOMBS` from `10` to `5`. This dynamically decreases the pre-allocated sprite pool size for bombs and fires (`POOL_FIRE = 85` instead of `170`), keeping the total DOM elements count after level load at ~390 (well below 500).
- **`tests/e2e/audit/audit.browser.spec.js`**: Updated the maximum DOM element budget assertions from `600` to `500` for both the `AUDIT-CI-09` budget test and the `AUDIT-B-03` pooling test.
- **`docs/implementation/audit-traceability-matrix.md`**: Updated the Last modified timestamp to keep verification synchronized.

### 🎯 Why
- **Rationale**: `AGENTS.md` mandates that the total number of DOM elements must be "≤ 500 total after level load." However, the E2E assertions were incorrectly expecting up to 600 elements, and the pre-allocated sprite pools were sized too generously for the 15x11 game board, resulting in 544 active DOM elements on boot. Sizing down the maximum concurrent bombs to 5 is a safe optimization that respects the budget.
- **Impact**: Codebase strictly conforms to the DOM elements constraints in `AGENTS.md`.
