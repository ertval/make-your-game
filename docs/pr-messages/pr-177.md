# 🚀 feat(A-07): deduplicate triggers and add fast-fail steps to policy gate workflow
> **Summary**: Deduplicated the CI workflow triggers (removed pull_request_review) and added separate fast-fail steps (linting, schema validation, and unit tests) before Playwright browsers installation.

---

## 📝 Description

### 🔄 What Changed
- [CI/CD]: Removed `pull_request_review` from `.github/workflows/policy-gate.yml` triggers.
- [CI/CD]: Added separate steps `Lint and Format Check`, `Validate JSON Schemas`, and `Run Unit Tests` in `.github/workflows/policy-gate.yml` before the Playwright installation/checks.

### 🎯 Why
- [Rationale]: To prevent duplicate workflow runs on the same commits. To allow the workflow to fail fast if basic syntax, schema validation, or unit tests fail, without waiting for the slow Playwright browser installation.
- [Impact]: Reduced CI resource consumption and faster feedback loop for developers.
