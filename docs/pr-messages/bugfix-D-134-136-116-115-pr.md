# 🔧 Track D: Resolve issues #136, #116, #115, #134 — Map & loader integrity

> **Summary**: Four Track D bug fixes — level-loader commit ordering, map-border integrity (all three levels), the `grid2D` mirror-write guard, and a HUD-completeness regression guard. #134 was already resolved by the prior HUD refactor; this PR locks it in with a test.

---

## 📝 Description

### 🔄 What Changed
- **`src/game/level-loader.js`** (#136 / BUG-23): `loadLevel` now commits `currentLevelIndex` and `world.setResource(mapResource)` **before** invoking the `onLevelLoaded` hook, so callbacks observe post-commit state.
- **`assets/maps/level-1.json`, `level-2.json`, `level-3.json`** (#115 / BUG-02): outer-border cells that were `DESTRUCTIBLE` set to `INDESTRUCTIBLE` — `[5][14]` on all three levels and `[5][0]` on level-3 (grid + `asciiBlueprint`).
- **`src/ecs/resources/map-resource.js`** (#115): `validateBorderIntegrity` tightened to reject **any** non-indestructible perimeter cell (previously permitted destructible).
- **`src/ecs/resources/map-resource.js`** (#116 / BUG-03): `setCell` drops the `if (map.grid2D[row])` guard so the 2D mirror stays in lockstep with the flat grid and genuine corruption surfaces instead of being silently swallowed.
- **Tests**: `tests/unit/game/level-loader.test.js` (#136), `tests/unit/resources/map-resource.test.js` (#115 + #116), `tests/e2e/map-border-integrity.spec.js` (#115), `tests/unit/systems/hud-system.test.js` (#134).

### 🎯 Why
- **BUG-23 (#136)**: Firing `onLevelLoaded` before the resource/index commit meant any callback reading `getResource('mapResource')` or `getCurrentLevelIndex()` saw the **old** level — a fragile event-ordering contract that invites stale-state reads on level loads.
- **BUG-02 (#115)**: A destructible border cell can be blown open by an explosion, leaving a visual hole while movement stays blocked (`getCell` clamps out-of-bounds to `INDESTRUCTIBLE`). Result: the player sees a hole but can't exit — a visual/gameplay desync. The ticket named level-3 only; the same defect existed on level-1 and level-2.
- **BUG-03 (#116)**: The guarded mirror write updated the flat grid but silently skipped `grid2D` when a row was missing, diverging the two representations with no signal — stale data consumed by row-iterating systems/tests.
- **BUG-21 (#134)**: The reported partial-HUD fallback path no longer exists; `hud-system` writes all six fields unconditionally and the no-adapter no-op lives in `hud-render-system`. Already fixed — this PR adds a guard so REQ-17 completeness can't regress.
