# 🚀 ci(A-07): deduplicate CI triggers and add fast-fail steps

> **Summary**: Optimizes the CI workflow by removing duplicate `pull_request_review` triggers and adding fast-fail steps (linting, schema validation, unit tests) before Playwright browser installation.

---

## 📝 Description

### 🔄 What Changed
- **`.github/workflows/policy-gate.yml`**:
  - Removed `pull_request_review` trigger to prevent duplicate workflows on review submission.
  - Inserted fast-fail steps for Biome check (`npm run check`), JSON schema validation (`npm run validate:schema`), and ECS unit tests (`npm run test:unit`) immediately after dependencies install and prior to the heavy Playwright browser download step.
- **`docs/implementation/audit-traceability-matrix.md`**: Updated the Last modified timestamp to keep verification synchronized.

### 🎯 Why
- **Rationale**: Triggering CI on both `pull_request` and `pull_request_review` causes duplicate, redundant workflow runs that waste developer time and runner resources. Furthermore, formatting, map schema, or unit test failures should reject the run immediately instead of waiting for the slow Playwright browser installation.
- **Impact**: Faster feedback loop for CI checks and reduced CI resource consumption.
