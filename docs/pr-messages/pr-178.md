# 🚀 feat(A-06): align DOM budget assertions with AGENTS.md limits and optimize sprite pool sizes
> **Summary**: Adjusted the E2E DOM element budget checks from 600 to 500 to align with `AGENTS.md` rules, and optimized pre-allocated sprite pool sizes (`POOL_MAX_BOMBS` from 10 to 5) to satisfy the budget constraint.

---

## 📝 Description

### 🔄 What Changed
- [Constants]: Changed `POOL_MAX_BOMBS` from `10` to `5` in `src/ecs/resources/constants.js`.
- [Tests]: Updated DOM count assertions from `toBeLessThanOrEqual(600)` to `toBeLessThanOrEqual(500)` in `tests/e2e/audit/audit.browser.spec.js`.

### 🎯 Why
- [Rationale]: `AGENTS.md` mandates `≤ 500 total after level load` for DOM elements. The E2E tests were checking against a relaxed budget of 600, which hid the fact that the total DOM element count was exceeding 500 (specifically, 544).
- [Impact]: Reduced pre-allocated DOM nodes in the sprite pool to comfortably fit under 500 elements (now ~454 elements) without affecting gameplay mechanics (5 concurrent bombs is plenty).
