# 🚀 feat(A-06): expose getWorld test hook and add E2E test for ghost staggered release timing
> **Summary**: Exposed the ECS `world` object on the window control hooks as `getWorld()`, and added a robust E2E test verifying ghost-house staggered release timings on Level 3. Closes #174.

---

## 📝 Description

### 🔄 What Changed
- [Bootstrap]: Added `getWorld: () => bootstrap.world` to the controls object in `src/main.ecs.js` to expose the ECS world interface.
- [Tests]: Added a new E2E test `AUDIT-F-13 ghost-house stagger and release timing conforms to requirements` in `tests/e2e/audit/audit.browser.spec.js`.
- [Tests]: Implemented manual reset of `ghostSpawnState` and ghost entity masks in the E2E test to clear boot-time level carryover and prevent clock desync flakes.

### 🎯 Why
- [Rationale]: The F-13 audit requires verifying the ghost-house release stagger timing contract (0s, 5s, 10s, 15s delay). While unit tests exist, browser E2E checks were missing. Exposing `getWorld()` is a prerequisites hook to properly inspect and isolate E2E state under test.
- [Impact]: Full E2E verification of F-13 stagger release rules in a headless browser, with robust clock tracking immune to VM execution speed.
