# 🔧 Track D: Remediation batch — issues #167, #163, #158, #154, #139, #156, #100

> **Summary**: Six Track D audit remediations — a sprite class-reset that no longer clobbers foreign classes (SEC-08), runtime grid cell-type range validation (SEC-04), board-sync adapter injection via the resource API (ARCH-02), a same-level-restart snapshot reset that kills ~165 redundant DOM writes (ARCH-06), removal of the dead legacy DOM renderer (DEAD-03), and an ownership-pattern gap fix in the policy gate (ARCH-04) — plus the #100 favicon enhancement (Ms. Ghostman icon: SVG primary + .ico/.png fallbacks). #126 (BUG-13) was independently verified already-fixed in `main` and is documented below (not re-fixed here). This is a `bugfix` branch so the cross-track favicon change — a Track D visual asset whose `<link>` tags must live in Track A's `index.html` — is carried under the ownership bypass.

---

## 📝 Description

### 🔄 What Changed
- **`src/ecs/systems/render-dom-system.js`** (#167 / SEC-08): the per-frame reset `el.className = 'sprite'` (which wiped **every** class on the pooled element) is replaced by `el.classList.remove(...MANAGED_SPRITE_CLASSES)` + `el.classList.add('sprite')`. `MANAGED_SPRITE_CLASSES` is a module-level, de-duplicated union built from the same source constants the apply-paths use (kind, player/fire/bomb frames, all ghost type × walk-frame combos, visual-flag states), so it can't drift. Foreign classes on a shared pool node now survive the reset.
- **`src/ecs/resources/map-resource.js`** (#163 / SEC-04): new `validateGridCellTypes` check wired into `validateMapSemantic`. Every grid cell must be an integer in `[0, MAX_CELL_TYPE_ID]` (derived from the `CELL_TYPE` enum). Out-of-range values (silently coerced by the `Uint8Array` flat grid and mis-handled by collision/pellet fallbacks) now fail closed at the load boundary.
- **`src/ecs/systems/board-sync-system.js`** (#154 / ARCH-02): the board adapter is now read via `context.world.getResource('boardAdapter')` instead of a closure parameter. Factory signature changed `createBoardSyncSystem(boardAdapter, options)` → `createBoardSyncSystem(options)`; `boardAdapter` added to `resourceCapabilities.read`.
- **`src/ecs/systems/board-sync-system.js`** (#158 / ARCH-06): on `context.renderFrame === 0` (restart / level-load) the stale snapshot is re-baselined to the fresh grid, so the diff no longer replays the previous run's mutations as ~165 redundant `updateCell` writes. Mirrors render-dom-system's frame-0 restart handling.
- **`src/game/bootstrap.js`** (#154/#158): registers `boardAdapter` as a world resource and constructs the board-sync system with the new options-only signature.
- **`src/adapters/dom/renderer-dom.js` + `tests/integration/adapters/renderer-dom.test.js`** (#139 / DEAD-03): deleted. The legacy `createDomRenderer` adapter threw if reached at runtime and was imported only by its own test; `src/main.ecs.js` comment updated to reflect the removal.
- **`scripts/policy-gate/lib/policy-utils.mjs`** (#156 / ARCH-04): added `board-sync-*` and `ghost-animation-*` source + test glob patterns to Track D's `TRACK_OWNERSHIP_RULES`, closing the ownership-mapping gap for systems that were already Track D in practice.
- **`index.html` + `public/favicon.svg` + `public/favicon.ico` + `public/favicon-32x32.png` + `public/favicon-16x16.png` + `public/apple-touch-icon.png`** (#100): add a Ms. Ghostman favicon. An SVG (declared first) is the crisp primary; the multi-size `.ico` and `.png`s are legacy fallbacks; `apple-touch-icon` covers iOS. Assets live in `/public` and are served at the site root by Vite in dev and build. Verified that real Chromium **and** WebKit (Safari's engine) fetch, decode, and rasterize all formats.
- **Tests**: `tests/unit/systems/render-dom-system.test.js` (#167 foreign-class preservation), `tests/unit/resources/map-resource.test.js` (#163 out-of-range/negative/non-integer rejection), `tests/unit/systems/board-sync-system.test.js` (#154 resource injection + #158 restart reset), `tests/integration/gameplay/d-08-render-dom-system.test.js` (mock element now provides `classList.remove`), `tests/e2e/favicon.spec.js` (#100 — asserts the favicon `<link>` tags are present and that each asset serves `200` with an `image/*` content-type).

### 🎯 Why
- **SEC-08 (#167)**: `el.className = 'sprite'` is a destructive write — any class added by another concern sharing the pooled DOM node (debug overlay, test marker, a future cross-cutting feature) was silently erased every frame. The targeted classList reset removes only the classes this system owns.
- **SEC-04 (#163)**: `validateMapSemantic` never range-checked cells. A value > 9 has no `CELL_TYPE` meaning yet is stored opaquely in the `Uint8Array`, corrupting pellet counts and collision. The JSON-schema mirror enforces this too, but it is skipped in the test env and is a separate source of truth — semantic validation is the always-on guard.
- **ARCH-02 (#154)**: AGENTS.md § Adapter Injection requires adapters be reached through the resource API. The closure-param path made board-sync inconsistent with every other adapter consumer and harder to test/audit.
- **ARCH-06 (#158)**: on same-level restart the dimensions are unchanged, so the old resize-triggered resync never fired; the stale (mutated) snapshot diffed against the freshly-repainted grid and re-emitted ~165 `updateCell` calls — a visible flash on constrained devices.
- **DEAD-03 (#139)**: dead code with a runtime-poison guard is pure liability; removing it shrinks the audit surface.
- **ARCH-04 (#156)**: without the glob patterns, ownership tooling couldn't attribute `board-sync-*` / `ghost-animation-*` to Track D, weakening the ownership gate.
- **#100 (favicon)**: the app shipped with no favicon, so browsers showed a generic/letter monogram in the tab. The SVG-first declaration gives a crisp icon at any tab size and sidesteps raster decode/size-selection ambiguity; the `.ico`/`.png` set covers browsers without SVG-favicon support.
